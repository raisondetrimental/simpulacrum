  management
  5. No testing infrastructure - No test directories or test files
  6. Backend structure too flat - All API logic in one massive file
  (excel_api.py)
  7. No proper requirements split - Should have dev/prod/test requirements
  8. Data directory exposed - Should not be in source control or should be in
  gitignore
  9. Generated files in source - JSON databases committed to git
  10. No CI/CD configuration - No GitHub Actions, pre-commit hooks, etc.

  11. Frontend lacks feature organization - Pages/components growing large

  read_dashboard.py vs excel_api.py
  13. No shared types/constants - Duplicated between frontend/backend
  14. Scripts directory underutilized - Only 2 scripts, should be organized

  15. Assets scattered - Images in multiple locations (etl/, web/public/,
